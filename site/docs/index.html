安装与配置

NoType 当前是一款面向 macOS 14+ 的菜单栏应用。当前公开下载的是未 notarize 的实验性构建，第一次使用时除了权限授权和配置外，还需要手动放行应用。

安装

  1. 打开下载页，进入最新 GitHub Release。
  2. 下载 `.dmg` 并把 `NoType.app` 拖入 `/Applications`。
  3. 首次双击时，如果 macOS 提示无法验证开发者，先不要删除应用。
  4. 到 `系统设置 -> 隐私与安全性` 里选择“仍要打开”，或在 Finder 中对应用执行 `Open`。
  5. 完成放行后，应用会出现在菜单栏。

权限授权

NoType 依赖两个系统权限：

  • `Microphone`：录制语音
  • `Accessibility`：把转写结果粘贴回当前输入框

如果 `Accessibility` 没有授权，NoType 可以继续完成转写，但无法稳定把文本注入到当前应用里。

Doubao 配置

在 `Settings` 里填写下面的字段：

  • `App ID`
  • `Resource ID`
  • `Access Token`

`Access Token` 会保存在 macOS Keychain，其他设置保存在本地 `UserDefaults`。

AI Rewrite

NoType 支持把口语稿整理成更适合发送或交给 AI 的文本。这个能力是可选的，不影响主转写链路。

  • `AI Rewrite Off`：直接插入 ASR 最终结果
  • `AI Rewrite On` 且配置完整：先改写，再插入
  • `AI Rewrite On` 但配置不完整：回退到原始转写

当前支持两条 provider 路径：`OpenAI-Compatible` 和 `Gemini`。

使用流程

  1. 在任意可输入区域聚焦。
  2. 按下默认快捷键 `Option + Space` 开始录音。
  3. 再次按下同一快捷键结束录音，或按 `Option + Esc` 取消。
  4. Doubao 返回最终文本后，NoType 会尝试把内容粘贴回当前焦点。
  5. 如果未检测到可编辑焦点，结果会保留在剪贴板里，供手动粘贴。

常见问题

按快捷键没反应

先确认 NoType 正在菜单栏运行，并且 `Accessibility` 已授权。

应用一打开就被 macOS 拦截

这是当前实验性构建的预期行为，因为它还没有经过 Apple notarization。按安装步骤手动放行即可。

能转写但没插入到当前应用

通常是当前应用焦点不可编辑，或 `Accessibility` 权限还没有生效。

打开了 AI Rewrite 但效果没变化

检查 provider、API key、model 是否已经填完整。配置缺失时，NoType 会继续使用原始转写结果，不会阻塞主链。

如果你更偏向开发者视角，可以直接查看仓库里的 README 和 scripts/ 下的打包脚本。
