A: Compact Dashboard

Core idea: Compress everything into 3 tight rows. Header fuses title + hotkey + status into one line. Language and LLM become side-by-side inline pills instead of dropdown menus.

Pros

  • Smallest footprint (~50% shorter)
  • Status glanceable at a glance
  • Hotkey subtle but visible

Cons

  • Tight — may feel cramped on first use
  • Less room for error/warning messages

B: Card-Based Groups

Core idea: Group related info into a visual card. Hotkey is the hero element (it's what you need to know). Language + LLM are side-by-side tiles inside the card. Clear hierarchy through background depth.

Pros

  • Clear visual hierarchy via card grouping
  • Hotkey prominent but not overwhelming
  • Feels native and polished

Cons

  • Slightly taller than A
  • Card nesting adds visual weight

C: Native Menu Style

Core idea: Ditch the "mini app" feel entirely. Centered icon + status, then standard macOS menu rows. Feels like a system utility, not a settings panel. Narrower width.

Pros

  • Most familiar to macOS users
  • Narrowest — least intrusive
  • Zero learning curve

Cons

  • Less visually distinctive
  • Centered icon may feel generic
